# T6 — Basketball Report Review (GitHub Pages)

Static, client-side version of the T6 human study. Hosts on any static-file
server (GitHub Pages, Netlify, S3, `python -m http.server`).

## Layout

```
github_pages/
├── index.html              landing page (User ID + email)
├── review.html             review interface (left: report, right: checkboxes)
├── css/style.css
├── js/review.js            checkbox state + localStorage + submit/download
├── data/samples_basketball.js  embedded sample data (regenerated by build_data.py)
├── build_data.py           regenerate samples_basketball.js from the parent JSON
└── setup.sh                run a local dev server on port 8765
```

## Local test

```bash
./setup.sh                # http://localhost:8765
```

## Deploying to GitHub Pages

1. Push the contents of `github_pages/` (or the whole repo) to GitHub.
2. In the repo settings, enable Pages and point it at the branch+folder that
   holds these files.
3. Set `FORMSPREE_URL` in `js/review.js` if you want automatic remote
   submission. Otherwise reviewers click **Download Responses (JSON)** and
   email you the file.

## Submission flow

- Per-item: each Submit&Next call writes the checked indices to
  `localStorage` under `t6_basketball_scores` so reviewers can resume.
- On Finish: payload is built from localStorage and either POSTed to
  `FORMSPREE_URL` (if set) or downloaded as JSON via the fallback button.

## Refreshing the data

After re-running `select_subset.py` in the parent folder:

```bash
python build_data.py
```

This regenerates `data/samples_basketball.js`. No other file needs to change.
